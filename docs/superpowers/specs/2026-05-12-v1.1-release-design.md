脚本结构：

```ts
import QRCode from 'qrcode';

const URL = process.env.QR_URL ?? 'https://hopkinson-bar.vercel.app';
const NAME = process.env.QR_NAME ?? 'qr-vercel';
const OUT_DIR = 'public';

await QRCode.toFile(`${OUT_DIR}/${NAME}.png`, URL, {
  width: 512, margin: 2, color: { dark: '#0A2540', light: '#FFFFFF' },
  errorCorrectionLevel: 'H',
});
await QRCode.toFile(`${OUT_DIR}/${NAME}.svg`, URL, { type: 'svg' });
console.log(`Generated QR codes for ${URL} → ${OUT_DIR}/${NAME}.{png,svg}`);
```

**首版不嵌入 Logo**：用 `errorCorrectionLevel: 'H'`（25% 容错）即可。若后续需要 Logo，再加 `sharp` 依赖在中心覆盖 64×64 PNG，本次不实现。

## D. BP / PID / CT 三块成品化

### D-1 BP-ANN 本构预测

#### 算法基础

- **网络结构**：单隐层 MLP。输入 3 维 `(应变 ε, 应变率 ε̇, 温度 T)`，隐层默认 12 个神经元（tansig 激活），输出 1 维 `应力 σ`（purelin 线性输出）。
- **训练**：Xavier 初始化 + 小批量 SGD + 学习率衰减 `lr / sqrt(1 + 0.02·epoch)`。
- **归一化**：输入与输出独立做 Min-Max 归一化，推理时反归一化。
- **训练样本来源**：
  1. 选定材料 `Material.stressStrainSample`（典型 10-20 点）作为基准；
  2. 用材料的 J-C 参数 + 随机扰动应变率 (×/÷ 2.5 倍) 与温度 (±50°C) 合成 `augmentFactor × 原始点数` 个样本。

#### 现状（工作树中已完成）

- `src/services/bpNetwork.ts` — BP 网络实现（前向、反向、Xavier、归一化、shuffle）。
- `src/services/bpShpbPredictor.ts` — 训练样本构造 + 训练入口。
- `src/features/material-analysis/BPPredictionPanel.tsx` — UI（隐层/epoch/学习率滑块 + loss 曲线 + 应力-应变三应变率泛化图）。

### D-2 PID 围压闭环伺服

#### 算法基础

- **被控对象**（`confiningPressureSimulator.ts`）：二阶带时滞系统
  ```
  τ² p̈ + 2ζτ ṗ + p = K u(t - Td) + d(t)
  ```
  - K=200 MPa（伺服阀全开稳态压力）、τ=0.35 s、ζ=0.45（欠阻尼，便于演示超调）、Td=0.06 s、噪声 σ=0.15 MPa。
- **控制器**（`pidController.ts`）：标准位置式 PID（Kp·e + Ki·∫e + Kd·de/dt），含输出饱和。
- **三档预设**：保守 / 经典 Z-N / 激进。
- **扰动**：t=3.5 s 注入 -10 MPa 持续 0.4 s。
- **性能指标**：超调 % / 稳态误差 / 2% 调节时间 / RMSE。

## E. Agent 审计

### 派遣方式

D 项三块代码提交后，**单条消息内并行派 3 个 `general-purpose` 子代理**（不传共享状态，每个 prompt 自包含）。

### Agent 任务模板

每个 agent 收到的 prompt 结构：

```
任务：审计 [模块名] 的物理/算法正确性。
背景：[论文引用 / 算法基础]
关注维度：[5-7 条具体维度]
输入：[相关文件绝对路径]
输出要求：
  - P0 = 必修（科学性错误 / 数据丢失 / 崩溃）
  - P1 = 应修（论文偏离 / 易误导用户的标签）
  - P2 = 建议（性能 / 可读性 / 文档）
  - 每个问题 ≤ 2 行：文件:行号 + 一句话描述 + 一句话建议
  - 报告总长 ≤ 250 字
```

### 三个 Agent 的具体维度

**Agent-BP** 审计 `bpNetwork.ts` + `bpShpbPredictor.ts` + `BPPredictionPanel.tsx`：

- Xavier 初始化方差是否正确（`sqrt(2/(rows+cols))`）
- tansig 导数是否用了 `1 - y²` 而非 `1 - tanh(x)²`（这里 y 已经是 tanh 输出，OK）
- SGD 更新方向是否正确（`-= lr/n * grad`）
- 归一化是否会在 max==min 时除零（有 `range > 0` 保护）
- 合成样本是否本质就是 J-C 复刻 → 文档是否如实声明
- R² 计算是否对多输出取平均

**Agent-PID** 审计 `pidController.ts` + `confiningPressureSimulator.ts` + `PIDServoPanel.tsx`：

- 是否处理了积分饱和（anti-windup）
- 二阶系统欧拉积分稳定性（dt=0.01s 是否够细）
- 时滞缓冲实现（环形缓冲 vs 简单 shift）
- Z-N 整定值是否符合 K=200/τ=0.35 的系统
- 性能指标公式：2% 调节时间定义是否标准
- `evalResponse` 中 settlingTime 当压力始终在带内时返回 0 还是 None

**Agent-CT** 审计 `ctFissureExtractor.ts` + `CTFissureExtractorPanel.tsx`：
