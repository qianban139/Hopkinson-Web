# 数智化电磁驱动霍普金森杆系统 — 实验全流程与页面联系

> 版本: v2.0 | 更新日期: 2026-03-18

---

## 1. 实验生命周期总览

完整实验从安全检查到报告导出共分为 **9 个阶段**，跨越 5 个功能模块。

```
Phase 0: 系统初始化
    ↓
Phase 1: 安全检查 ─────────── SystemMonitor (/monitor)
    ↓
Phase 2: 材料选择与参数配置 ── VirtualLab (/lab)
    ↓
Phase 3: 实验执行 ─────────── VirtualLab (/lab)
    ↓
Phase 4: 数据采集 ─────────── VirtualLab (/lab) → DataBus
    ↓
Phase 5: AI优化分析 ────────── AIControl (/ai)
    ↓
Phase 6: 优化参数回写 ──────── AIControl → DataBus → VirtualLab
    ↓
Phase 7: 多场耦合分析 ──────── MultiField (/multifield)
    ↓
Phase 8: 材料力学分析 ──────── MaterialAnalysis (/analysis)
    ↓
Phase 9: 报告导出 ─────────── MaterialAnalysis (/analysis)
```

## 2. 各阶段详细说明

### Phase 0: 系统初始化

**触发条件**: 用户访问网站
**所在页面**: 全局

**流程**:
1. Zustand store 初始化（`useAppStore`、`useExperimentDataBus`）
2. 创建实验会话 `experimentSession`，生成唯一 `session-{timestamp}` ID
3. 初始化安全检查清单（7项默认 `pending`）
4. 设置 `currentPhase: 'setup'`
5. 初始化 `pendingModules: ['monitor', 'lab', 'ai', 'multifield', 'analysis']`

**输出数据**:
```typescript
experimentSession: {
  id: 'session-1710700000000',
  startedAt: 1710700000000,
  completedModules: [],
  pendingModules: ['monitor', 'lab', 'ai', 'multifield', 'analysis'],
  currentPhase: 'setup'
}
```

### Phase 1: 安全检查

**触发条件**: 用户进入 SystemMonitor 页面，或尝试进入 VirtualLab 时弹出强制检查
**所在页面**: `/monitor`
**WorkflowProgressBar状态**: 安全检查 ● (当前)

**流程**:
1. 显示 7 项安全检查清单：
   - 电容组状态（检查充放电和漏电流）
   - 冷却系统（验证冷却液流量和基线温度）
   - EMI屏蔽（确认屏蔽完整性，背景EMI<50dB）
   - 试件对位（验证杆件对位和试件安装）
   - 数据采集（验证应变片连接和采样率设置）
   - 紧急系统（测试紧急停止响应时间）
   - 人员安全（确认安全区域清空和PPE检查）
2. 每项执行模拟传感器读数检测
3. 状态流转：`pending` → `checking` → `pass`/`warning`/`fail`
4. 全部通过后调用 `completeSafetyChecklist()`

**状态变更**:
```typescript
// completeSafetyChecklist() 执行后:
safetyChecklistCompleted: true
safetyChecklistTimestamp: Date.now()
experimentSession.completedModules: ['monitor']
experimentSession.currentPhase: 'safety'
```

**解锁条件**: `safetyChecklistCompleted === true` 后 VirtualLab 解锁

---

### Phase 2: 材料选择与参数配置

**触发条件**: 安全检查通过后，用户进入 VirtualLab
**所在页面**: `/lab`
**WorkflowProgressBar状态**: 安全检查 ✓ → 虚拟实验 ● (当前)

**流程**:
1. **材料选择**（左侧面板）：
   - 浏览 7 大类材料（金属、岩石、混凝土、陶瓷、聚合物、泡沫、复合材料）
   - 搜索或分类筛选
   - 选中材料 → `useAppStore.setSelectedMaterial()`
   - 显示材料基本属性（密度、弹性模量、屈服强度）

2. **参数配置**（右侧面板）：
   - 选择预设或手动调节：
     - 标准SHPB: 280V / 25kA / 500μs
     - 高应变率: 350V / 45kA / 300μs
     - 岩石冲击: 320V / 35kA / 800μs
     - 低速测试: 220V / 15kA / 1100μs
   - 调节电压(220-350V)、电流(0-50kA)、脉宽(200-1100μs)
   - 选择波形类型（正弦/方波/三角/脉冲）
   - 可选启用围压（XYZ三轴，0-200MPa）
   - 查看AI推荐值（紫色徽章），一键应用

3. **AI辅助参数设置**（可选）：
   - 点击"让AI设置参数"按钮
   - AI根据材料类型推荐最优参数组合
   - 用户确认后应用

**关键数据**:
```typescript
experimentParams: {
  voltage: 280,       // V
  current: 25000,     // A
  pulseWidth: 500,    // μs
  waveform: '梯形',
  materialId: 'metal-01'
}
```

### Phase 3: 实验执行

**触发条件**: 用户点击"开始实验"按钮
**所在页面**: `/lab`

**流程（2D动画状态机 7阶段）**:

| 阶段 | 名称 | 持续时间 | 视觉效果 |
|------|------|----------|----------|
| 1 | `idle` | — | 设备静态，电容微弱呼吸发光 |
| 2 | `charging` | ~2s | 电容液位上升，电压读数递增，端子火花 |
| 3 | `coilAccel` | ~1.5s | 三级线圈依次发光，磁场线动画 |
| 4 | `strikerLaunch` | ~1s | 子弹加速+运动模糊，速度标注 |
| 5 | `wavePropagate` | ~2s | 应力波沿杆件传播，色彩可视化 |
| 6 | `deformation` | ~1.5s | 试样变形（韧性鼓胀/脆性裂纹） |
| 7 | `dataCollect` | ~1.5s | DAQ信号采集，波形逐点构建 |

**实时数据生成**:
- 入射波 σᵢ(t)、反射波 σᵣ(t)、透射波 σₜ(t) 三通道波形
- 波形幅值由参数（电压、材料刚度、阻尼）计算
- ECharts 实时绘制，与2D动画同步

---

### Phase 4: 数据采集与发布

**触发条件**: 实验动画完成（`dataCollect` 阶段结束）
**所在页面**: `/lab`

**流程**:
1. 实验完成回调 `handleExperimentComplete()` 触发
2. 组装 `LabExperimentResult` 数据包：
   ```typescript
   {
     materialName: '45号钢',
     materialCategory: 'metal',
     params: { voltage: 280, current: 25000, pulseWidth: 500 },
     waveformData: { incident: [...], reflected: [...], transmitted: [...] },
     peakStress: 850,    // MPa
     strainRate: 2500,   // /s
     timestamp: Date.now()
   }
   ```
3. 调用 `useExperimentDataBus.publishLabExperiment(data)`
4. DataBus 自动:
   - 存储到 `lastLabExperiment`
   - 记录数据流日志
   - 更新会话：`completedModules` 添加 `'lab'`，`currentPhase` → `'experiment'`

**数据流向**:
```
VirtualLab → DataBus.lastLabExperiment
                ├── → AIControl (读取进行优化)
                ├── → MultiField (读取基准数据)
                └── → MaterialAnalysis (读取分析数据)
```

### Phase 5: AI优化分析

**触发条件**: 用户进入 AIControl 页面，或AI自动触发优化
**所在页面**: `/ai`
**WorkflowProgressBar状态**: 虚拟实验 ✓ → AI优化 ● (当前)

**流程**:
1. 读取 DataBus 中的 `lastLabExperiment` 数据
2. 显示"数据来源: Lab ✓"状态标识
3. 执行三级优化管线：

| 步骤 | 算法 | 输入 | 输出 | 耗时 |
|------|------|------|------|------|
| 1 | LSTM时序预测 | 波形时序数据 | 预测波形趋势 | ~1s |
| 2 | WGAN-GP生成 | 预测结果+噪声 | 优化目标波形 | ~1s |
| 3 | PPO策略优化 | 目标波形+参数空间 | 最优参数组合 | ~1s |

4. 优化状态流转：`idle` → `lstm` → `wgan` → `ppo` → `complete`
5. 生成优化结果：
   ```typescript
   aiOptimizedParams: {
     voltage: 320,      // V (原280V)
     current: 35000,    // A (原25000A)
     pulseWidth: 650,   // μs (原500μs)
     improvements: {
       stressUniformity: 23,    // +23%
       energyEfficiency: 15,    // +15%
       peakAccuracy: 18,        // +18%
       waveformQuality: 12      // +12%
     },
     timestamp: Date.now()
   }
   ```

**可视化输出**:
- 管线节点实时状态动画
- 优化前后参数对比面板
- AI决策洞察卡片（解释 WHY）
- 原始波形 vs AI优化波形对比图
- 优化历史时间线

---

### Phase 6: 优化参数回写

**触发条件**: 用户点击"应用优化参数"按钮
**所在页面**: `/ai`

**流程**:
1. 调用 `publishAIOptimization(params)` 写入 DataBus
2. DataBus 自动追加到 `aiOptimizationHistory`（保留最近20条）
3. 记录数据流日志
4. VirtualLab 页面通过 Zustand selector 实时监听变化
5. VirtualLab 参数面板显示 AI 推荐值徽章（紫色）
6. 用户可在 VirtualLab 一键应用推荐参数
7. 可选：用优化参数重新执行实验（回到Phase 3）

**数据流向**:
```
AIControl → DataBus.aiOptimizedParams → VirtualLab (AI推荐值徽章)
```

---

### Phase 7: 多场耦合分析

**触发条件**: 用户进入 MultiField 页面
**所在页面**: `/multifield`
**WorkflowProgressBar状态**: AI优化 ✓ → 多场耦合 ● (当前)

**流程**:
1. 读取 DataBus 中的 `lastLabExperiment` 作为基准数据
2. 选择工程场景预设：
   - 深部矿井（高温高压）
   - 航空航天（极端温度循环）
   - 核反应堆（辐射+高温）
   - 电动汽车电池碰撞（高应变率+电磁场）
3. 配置三场参数：
   - 温度场：20-1000°C
   - 应力场：0-2000MPa
   - 电磁场：0-100T
4. 执行耦合分析，计算6种耦合效应：
   - 热膨胀（T→σ）
   - 绝热生热（σ→T）
   - 洛伦兹力（B→σ）
   - 磁弹效应（σ→B）
   - 焦耳热（B→T）
   - 居里温度效应（T→B）
5. 生成耦合结果并发布到 DataBus

**关键输出 — 结果对比面板**:
- 蓝线: 标准SHPB曲线（来自Lab基准数据）
- 红线: 多场耦合修正曲线
- 量化偏差注释：热软化偏差 X%、涡流能耗 Y%、总体偏差 Z%
- 核心信息: "忽略耦合效应，你的预测会偏差X%"

**数据流向**:
```
DataBus.lastLabExperiment → MultiField (基准数据)
MultiField → DataBus.lastMultiFieldExperiment → MaterialAnalysis
```

### Phase 8: 材料力学分析

**触发条件**: 用户进入 MaterialAnalysis 页面
**所在页面**: `/analysis`
**WorkflowProgressBar状态**: 多场耦合 ✓ → 材料分析 ● (当前)

**流程**:
1. 顶部实验数据集成横幅显示上次实验信息
2. 5个分析Tab:

| Tab | 功能 | 数据来源 |
|-----|------|----------|
| 应力-应变曲线 | 多材料叠加对比 | Lab波形数据 + 材料库 |
| J-C拟合 | Johnson-Cook模型参数可视化 | 材料库J-C参数 |
| 性能雷达 | 多维性能雷达图 | 材料库属性 |
| AI预测 | 输入应变率+温度预测响应 | AI模型推理 |
| 参数总表 | 全材料J-C参数表 | 材料库 |

3. 对比模式：最多4种材料同时对比
4. AI预测：输入应变率和温度条件，预测动态力学响应

## 3. 模块间数据流 Schema

### 3.1 数据总线架构

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│ SystemMonitor │     │  VirtualLab  │     │  AIControl   │
│   (/monitor) │     │    (/lab)    │     │    (/ai)     │
└──────┬───────┘     └──────┬───────┘     └──────┬───────┘
       │                    │                    │
       │ safetyChecklist    │ labExperiment      │ aiOptimization
       │ Completed          │ Result             │ Params
       ▼                    ▼                    ▼
┌──────────────────────────────────────────────────────────┐
│                    DataBus (Zustand)                      │
│                                                          │
│  lastLabExperiment    │  aiOptimizedParams                │
│  lastMultiFieldExp    │  aiOptimizationHistory            │
│  safetyChecklist      │  dataFlowLog                     │
│  experimentSession    │                                   │
└──────────────────────────────────────────────────────────┘
       │                    │                    │
       │                    │ labExperiment      │ couplingResult
       │                    │ + multiField       │
       ▼                    ▼                    ▼
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  MultiField  │     │  Material    │     │ DataBus      │
│ (/multifield)│     │  Analysis    │     │ Visualization│
└──────────────┘     │ (/analysis)  │     └──────────────┘
                     └──────────────┘
```

### 3.2 数据流边定义

| # | 源模块 | 目标模块 | 数据类型 | 说明 |
|---|--------|----------|----------|------|
| 1 | SystemMonitor | VirtualLab | `safetyChecklistCompleted` | 安全检查通过后解锁实验 |
| 2 | VirtualLab | AIControl | `LabExperimentResult` | 实验波形数据供AI分析 |
| 3 | VirtualLab | MultiField | `LabExperimentResult` | 基准数据供耦合对比 |
| 4 | VirtualLab | MaterialAnalysis | `LabExperimentResult` | 实验数据供深度分析 |
| 5 | AIControl | VirtualLab | `aiOptimizedParams` | AI优化参数回写 |
| 6 | MultiField | MaterialAnalysis | `MultiFieldExperimentResult` | 耦合数据供综合分析 |

### 3.3 核心数据类型

```typescript
// 实验结果
interface LabExperimentResult {
  materialName: string;
  materialCategory: string;
  params: {
    voltage: number;      // V
    current: number;      // A
    pulseWidth: number;   // μs
  };
  waveformData: {
    incident: number[];   // 入射波 σᵢ(t)
    reflected: number[];  // 反射波 σᵣ(t)
    transmitted: number[];// 透射波 σₜ(t)
  };
  peakStress: number;     // MPa
  strainRate: number;     // /s
  timestamp: number;
}

// 多场耦合结果
interface MultiFieldExperimentResult {
  thermalParams: { temperature: number; gradient: number };
  mechanicalParams: { stress: number; strainRate: number };
  electromagneticParams: { fieldStrength: number; frequency: number };
  couplingResults: {
    thermalMechanical: number;
    mechanicalEM: number;
    thermalEM: number;
    totalCoupling: number;
  };
  timestamp: number;
}

// AI优化参数
interface AIOptimizedParams {
  voltage: number;
  current: number;
  pulseWidth: number;
  improvements: Record<string, number>;
  timestamp: number;
}

// 实验会话
interface ExperimentSession {
  id: string;
  startedAt: number;
  completedModules: string[];
  pendingModules: string[];
  currentPhase: ExperimentPhase;
}

type ExperimentPhase = 'setup' | 'safety' | 'experiment'
  | 'optimization' | 'coupling' | 'analysis' | 'complete';
```

## 4. AI引导式工作流

### 4.1 全自动实验流程（语音/文本指令）

用户说："帮我用Q235钢做一个280V的标准SHPB压缩实验"

AI编排引擎执行序列：

```
Step 1: navigate.toPage → /monitor
Step 2: monitor.runSafetyCheck → 执行7项检查
  → 等待完成...
Step 3: navigate.toPage → /lab
Step 4: lab.selectMaterial → { name: 'Q235钢' }
Step 5: lab.setVoltage → { value: 280 }
Step 6: lab.setCurrent → { value: 25000 }
Step 7: lab.setPulseWidth → { value: 500 }
Step 8: lab.startExperiment → 启动实验
  → 等待动画完成...
Step 9: navigate.toPage → /ai
Step 10: ai.startOptimization → 执行三级优化
  → 等待完成...
Step 11: navigate.toPage → /analysis
Step 12: analysis.selectMaterial → { name: 'Q235钢' }
```

### 4.2 操作可视反馈

每个步骤执行时：
1. **高亮环**: 目标UI元素出现青色脉冲边框（通过 `data-ai-target` 定位）

3. **操作日志**: AI面板内时间线记录所有操作状态
4. **语音播报**: TTS播报当前操作（"正在选择Q235钢材料"）

### 4.3 AI可执行动作注册表

## 5. 错误恢复流程

### 5.1 安全检查失败

```
安全检查项失败
  ├── 单项失败(warning): 显示警告，允许继续（降级运行）
  ├── 单项失败(fail): 标红，阻止通过
  │     └── 用户修复后重新检查该项
  └── 全部重检: 点击"重新检查"按钮
        └── resetSafetyChecklist() → 重置所有项为 pending
```

### 5.2 实验执行中断

```
实验动画异常中断
  ├── Canvas渲染错误: 自动回退到 idle 状态
  ├── 用户手动停止: 动画暂停，保留当前帧
  └── 紧急停止:
        ├── MonitorStrip 紧急停止按钮触发
        ├── 全局告警（红色闪烁）
        └── 所有参数归零，回到 idle
```

### 5.3 AI优化失败

```
AI优化异常
  ├── LLM API超时: 显示错误提示，允许重试
  ├── 无实验数据: 显示"请先运行实验"引导
  └── 优化结果异常:
        ├── 参数超出安全范围 → 自动裁剪到有效区间
        └── 置信度过低(<50%) → 警告提示，建议重新实验
```

### 5.4 数据流断裂

```
模块间数据不可用
  ├── Lab数据缺失:
  │     ├── AIControl: 显示"尚无实验数据"提示
  │     ├── MultiField: 使用默认基准参数
  │     └── MaterialAnalysis: 仅显示材料库数据
  └── MultiField数据缺失:
        └── MaterialAnalysis: 跳过耦合对比，仅显示标准分析
```

```
                    ┌─────────┐
                    │  setup  │ ← 初始状态

                         │ completeSafetyChecklist()
                    ┌────▼────┐
                    │ safety  │
                    └────┬────┘
                         │ publishLabExperiment()
                 ┌───────▼────────┐
                 │   experiment   │
                 └───────┬────────┘
                         │ updateSessionPhase('optimization')
                ┌────────▼─────────┐
                │  optimization    │
                └────────┬─────────┘
                         │ publishMultiFieldExperiment()
                  ┌──────▼───────┐
                  │   coupling   │
                  └──────┬───────┘
                         │ updateSessionPhase('analysis')
                  ┌──────▼───────┐
                  │   analysis   │
                  └──────┬───────┘
                         │ updateSessionPhase('complete')
                  ┌──────▼───────┐
                  │   complete   │
                  └──────────────┘
```

## 7. 全局组件与页面联系

### 7.1 WorkflowProgressBar

- 固定在导航栏下方（`top: 64px`）
- 自动检测 DataBus 中各模块完成状态
- 检测逻辑：
  - 安全检查: `safetyChecklistCompleted === true`
  - 虚拟实验: `lastLabExperiment !== null`
  - AI优化: `aiOptimizedParams !== null`
  - 多场耦合: `lastMultiFieldExperiment !== null`
  - 材料分析: 由用户手动确认或导出触发
- 首页(`/`)隐藏

### 7.2 MonitorStrip

- 固定在页面底部
- 实时显示5项关键指标（电压、电流、储能、温度、EMI）
- 安全检查状态图标（盾牌）
- 超阈值时变色警告
- 展开详细视图，链接到 `/monitor`

### 7.3 DataBusVisualization

- 显示在 AIControl 页面
- SVG 5节点拓扑（Monitor、Lab、AI、MultiField、Analysis）
- 6条数据边，活跃边显示流动光点动画
- 活跃状态由 DataBus 数据存在性决定
- 节点可点击跳转到对应模块

### 7.4 AIAmbientGlow

- AI助手激活时显示
- 左上角青色 + 右下角紫色径向渐变
- 呼吸动画（scale + opacity）
- `pointer-events-none` 不影响交互

### 7.5 AICommandCenter

- 右下角悬浮球触发
- 支持文本和语音输入
- 操作日志实时显示
- 跨页面操作分发（通过 Zustand + CustomEvent）
