## 一、设计背景与目标

### 1.1 当前痛点

| 维度 | 现状 | 问题 |
|------|------|------|
| 平台定位 | 定位为「软硬结合智能实验平台」 | 实际仅为教学仿真工具，无硬件接口 |
| AI 助手 | 基础对话 + 简单语音 | 缺乏 Agent 自主执行、RAG 知识库、深度多模态 |
| 页面结构 | 7 个页面平铺 | 部分页面功能重叠，定位模糊 |
| 数据闭环 | 仿真数据自循环 | 无真实设备接入，无实验数据沉淀 |
| 材料分析 | 弱于虚拟实验室 | 仅展示，缺乏深度分析能力 |

### 1.2 升级目标

1. **平台跃升**：从「教学仿真工具」→「软硬结合智能实验平台」
2. **AI 跃升**：从「基础聊天助手」→「类小爱/小艺的科研伙伴」
3. **结构优化**：从「7 页平铺」→「核心 5 模块 + 强联动数据闭环」
4. **能力跃升**：从「单一仿真」→「真实设备 + 真实数据 + AI 智能」
5. **闭环形成**：物理实验 → 真实数据 → AI 分析 → 反馈优化

### 1.3 升级原则

- **真实优先**：所有指标和数据可对接真实硬件，仿真作为补充
- **AI 渗透**：AI 助手不是孤立功能，而是渗透到每个页面的伙伴
- **闭环驱动**：每个模块的输出必须能流入下一个模块
- **教学兼顾**：保留小白模式，但不能拖累专业能力

## 二、平台核心定位重申

### 2.1 一句话定位

> **数智化电磁驱动霍普金森杆多场耦合动态测试系统** ——
> 一体式（实验—数据分析—实验报告）的科研学习与数据分析智能实验室。

### 2.2 与传统平台的差异化

| 维度 | 纯仿真平台 | 纯硬件控制软件 | 本平台 |
|------|----------|-------------|--------|
| 真实设备连接 | ❌ | ✅ | ✅ |
| AI 智能辅助 | 弱 | 无 | 强 |
| 教学功能 | 强 | 无 | 强 |
| 远程协作 | 部分 | 无 | 强 |
| 数据闭环 | 自循环 | 单向 | 物理↔虚拟双向闭环 |
| 端到端报告 | ❌ | ❌ | ✅ |

### 2.3 价值主张

```
教学价值：让初学者通过 AI 引导安全地完成完整实验
科研价值：让研究员快速缩小参数搜索范围，AI 辅助处理弥散性、应变率不恒定等痛点
工程价值：实测数据 + AI 算法生成可发表级实验报告
```

### 2.4 关键差异化能力（必须强化）

1. **应力波弥散性自动校正**（AI 信号处理）
2. **应变率不恒定的智能补偿**（AI 算法）
3. **多组实验跨实验学习**（深度学习）
4. **真实数据 vs 仿真预测的双向校验**
5. **基于真实数据训练的领域知识库**

## 三、用户画像与典型场景

### 3.1 用户 A：科研人员

**画像**：材料力学/动力学领域的研究员、博士生、工程师

**核心需求**：
- 测量某种试样在多种应变率/温度/围压下的动态力学性质
- 处理应力波弥散、应变率不恒定等数据问题
- 拟合本构模型参数
- 生成可发表级实验报告

**典型场景**：
```
场景：研究员张老师测试一种新型高熵合金在 1000-3000/s 应变率下的动态行为

1. 唤醒「小智」：「小智，我要测一种新型高熵合金，应变率范围 1000-3000」
2. AI 询问材料参数（密度、弹性模量、估计屈服强度）
3. AI 推荐参数组合（电压、脉宽、试样尺寸）
4. 安全检查通过 → 真实设备启动
5. 实验执行，AI 实时监控应力平衡
6. 数据采集 → AI 自动校正弥散性 → 应变率恒定性补偿
7. 自动进入材料分析页 → AI 拟合 J-C 参数
8. 生成完整实验报告（含原理、数据、图表、结论、可复现参数）
9. 一键导出 PDF + CSV + JSON
```

**关键能力需求**：
- ✅ AI 真实数据分析能力
- ✅ 信号处理算法（弥散校正）
- ✅ 跨实验学习
- ✅ 端到端报告生成
- ✅ 真实设备接口

### 3.2 用户 B：学生/初学者

**画像**：高校本科生、研究生新生、对动态力学感兴趣的爱好者

**核心需求**：
- 学习霍普金森杆原理与实验过程
- 在零风险环境下完成完整实验
- 理解每个步骤的物理意义
- 获得直观的可视化反馈

**典型场景**：
```
场景：本科生小李第一次接触 SHPB 实验

1. 唤醒「小智」：「小智，我想学习霍普金森杆」
2. AI 切换为教学模式：「好的，我先带你看一下装置结构」
3. 引导浏览 3D 数字孪生 → 解释每个部件
4. 进入虚拟实验室仿真模式
5. AI 一步步引导：「现在选择一个你想测的材料，建议从铝合金开始」
6. 每一步都解释原理：「为什么选 2000V？因为对应应变率约 1500/s」
7. 观察实验动画 → 实时讲解三波形成机制
8. 完成后引导查看应力-应变曲线 → 解释每个阶段含义
9. 推荐相关测验和拓展知识

特点：纯仿真模式，零硬件成本，可重复练习
```

**关键能力需求**：
- ✅ AI 教学引导能力
- ✅ 直观 2D/3D 可视化
- ✅ 知识图谱关联
- ✅ 仿真模式
- ✅ 在线测验

### 3.3 跨场景共性能力

| 能力 | A 用户 | B 用户 |
|------|--------|--------|
| AI 助手 | 科研伙伴 | 资深教师 |
| 实验执行 | 真实设备 | 仿真模式 |
| 数据分析 | 深度算法 | 基础解释 |
| 报告生成 | 学术级 | 学习总结 |
| 安全检查 | 简化（专家） | 详细引导 |

→ 这要求 AI 助手能**根据用户身份自动切换语气和深度**。

### 5.3 五大核心子系统

#### 5.3.1 子系统 A：多模态交互引擎

**目标**：让用户用最自然的方式与小智交互

**能力清单**：
1. **语音交互**：
   - 实时语音唤醒（已有 Porcupine）
   - 连续对话（无需重复唤醒）
   - 自然中断（用户开口即打断 AI）
   - 双语支持（中文 + 英文）
   - 实时字幕显示
2. **文字输入**：
   - 富文本输入框
   - 数学公式支持（LaTeX）
   - 拖放文件
3. **图像/视频**：
   - 上传实验照片 → 自动识别试样和损伤
   - 上传示波器截图 → 自动提取波形数据
   - 视频帧分析（裂纹扩展速度）
4. **传感器数据**：
   - 实时拖入波形文件 → 自动诊断
   - 拖入应变片 CSV → 自动绘图

**实现要点**：
- 现有 `realtimeVoicePipeline.ts` 升级为支持中断
- 新增 `multimodalProcessor.ts` 处理图像/文件
- LLM 调用切换为支持视觉的模型（GPT-4o / Claude 3.5 Sonnet / GLM-4V）

#### 5.3.2 子系统 B：Agent 自主执行引擎

**目标**：让小智能拆解复杂任务并自主完成

**能力清单**：
1. **任务规划**：
   - 接收高层目标（"测一下这个材料的动态屈服强度"）
   - 自动拆解为子步骤（选材料 → 设参数 → 安检 → 执行 → 分析）
   - 显示规划过程让用户审查
2. **工具调用**：
   - Function Calling 接入平台所有 API
   - 调用物理引擎、优化引擎
   - 调用真实设备控制接口
   - 调用外部 API（材料数据库、文献检索）
3. **多 Agent 协作**：
   - **Orchestrator**：任务分配
   - **Scientist**：实验方案设计
   - **Engineer**：参数优化
   - **Executor**：实际执行
   - **Analyst**：结果分析
4. **闭环优化**：
   - 实验失败 → AI 分析原因 → 自动调整
   - 跨实验学习

**架构**：

```
用户高层目标
    │
    ▼
┌─────────────┐
│ Orchestrator │ ─── 任务分配
└──────┬──────┘
       │
   ┌───┴───┬────────┬─────────┐
   ▼       ▼        ▼         ▼
┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│Scien.│ │Engin.│ │Exec. │ │Analy.│
│方案   │ │参数   │ │执行  │ │分析  │
└──────┘ └──────┘ └──────┘ └──────┘
   │       │        │         │
   └───────┴────────┴─────────┘
                │
                ▼
         结果反馈给 Orchestrator
                │
                ▼
            用户呈现
```

**实现要点**：
- 新增 `src/features/ai-assistant/agent/` 目录
- 实现 `agentOrchestrator.ts` 主控
- 实现 5 个角色 Agent：`scientistAgent.ts`、`engineerAgent.ts` 等
- 工具注册表 `toolRegistry.ts`，注册所有可调用工具

#### 5.3.4 子系统 D：科学实验专属能力

**目标**：让小智成为真正的实验助手，而不是通用聊天机器人

**特化能力**：

1. **应力波弥散校正**
   - 用户上传原始波形 → AI 自动识别弥散
   - 调用 Pochhammer-Chree 修正算法
   - 输出校正后波形 + 处理报告
2. **应变率恒定性补偿**
   - 检测应变率漂移
   - 自动建议波形整形器参数
   - 或对结果做后处理修正
3. **应力平衡判据**
   - 自动计算 R = |σ₁-σ₂|/max(σ₁,σ₂)
   - 红黄绿三级标识
   - 不达标时给出原因和建议
4. **本构参数自动拟合**
   - 多组实验数据 → 自动拟合 J-C/Cowper-Symonds/Z-A
   - 给出拟合优度 R²
   - 推荐最佳模型
5. **跨实验学习**
   - 历史实验数据训练专属小模型
   - 新材料预测先验
   - 持续微调

**实现要点**：
- 新增 `src/services/signalProcessing/` 目录
- `dispersionCorrection.ts`、`strainRateCompensation.ts`、`stressEquilibrium.ts`
- `constitutiveFitting.ts` — 通用拟合算法
- 这些算法不仅供 AI 调用，也供用户直接使用

#### 5.3.5 子系统 E：专家级输出与报告生成

**目标**：让小智的输出达到专业级水准

**输出形式**：

## 七、虚拟实验室双模式精确设计

### 7.1 设计理念

虚拟实验室是平台的**核心入口**，它不仅是仿真工具，更是真实设备的**数字操作台**。

### 7.2 双模式架构

```
┌────────────────────────────────────────────────────┐
│              虚拟实验室 /lab                         │
├────────────────────────────────────────────────────┤
│                                                    │
│  ┌──────────────┐         ┌──────────────┐       │
│  │  仿真模式     │         │   连接模式     │       │
│  │ (Simulation) │   ⇄    │  (Connected)  │       │
│  └──────────────┘         └──────────────┘       │
│                                                    │
│  数据源：物理引擎计算       数据源：真实设备 WebSocket│
│  适用：教学、参数探索       适用：科研、真实数据采集  │
│                                                    │
└────────────────────────────────────────────────────┘
```

模式由页面顶部 Tab 切换，**所有 UI 组件保持一致**，仅数据源不同。

### 7.3 仿真模式 (Simulation Mode)

**特征**：
- 数据源：`shpbPhysicsEngine.runSHPBSimulation()`
- 无需硬件，完全离线可用
- 适合教学、参数探索、AI 优化

**UI 元素**：

```
┌──────────────────────────────────────────────────────┐
│ [仿真模式] 切换到连接模式                              │
├─────────────┬────────────────────────────────────────┤
│             │                                        │
│  参数面板    │       3D/2D 装置可视化                  │
│             │       (动画播放)                       │
│  - 材料选择  │                                        │
│  - 电压      │                                        │
│  - 脉宽      │                                        │
│  - 温度      │                                        │
│  - 围压(XYZ) │                                        │
│  - EMI 场    │                                        │
│             │                                        │
│  [AI 优化]   │                                        │
│  [开始实验]   │                                        │
│             │                                        │
├─────────────┴────────────────────────────────────────┤
│        实时三波波形 (入射/反射/透射)                   │
├──────────────────────────────────────────────────────┤
│   ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐               │
│   │峰值σ │ │应变率│ │吸收能│ │平衡R │  关键指标      │
│   └──────┘ └──────┘ └──────┘ └──────┘               │
├──────────────────────────────────────────────────────┤
│        [跳转到材料分析]  [生成报告]  [保存数据]          │
└──────────────────────────────────────────────────────┘
```

**操作流程**：
1. 选择材料
2. 设置参数（含可选的多场耦合参数：温度、围压、EMI）
3. 可选：点击「AI 优化」让 AI 自动找最优参数
4. 安全检查（仿真模式下简化）
5. 「开始实验」→ 播放动画 → 显示结果
6. 跳转到材料分析或生成报告

### 7.4 连接模式 (Connected Mode)

**特征**：
- 数据源：本地后端 WebSocket
- 需要后端服务运行
- 适合科研、真实数据采集

**UI 元素**：

与仿真模式**完全相同的布局**，但增加：

```
顶部状态条变化：
- 当前模式：连接模式
- 设备连接状态：✅ 已连接 / ❌ 未连接
- 后端服务状态：✅ 运行中 / ❌ 未运行
- 最近一次心跳：1.2s 前

参数面板变化：
- 「读取设备当前参数」按钮
- 「下发参数到设备」按钮
- 「读取波形文件」按钮（导入示波器/DAQ 数据）

实时波形变化：
- 真实采样频率（标注 1MSa/s 等）
- 来源标记：「来自 DAQ-001」
- 时间戳精确到微秒
```

**操作流程**：
1. 检查设备连接（顶部状态条）
2. 读取设备当前参数 / 手动设置参数
3. 通过 AI 助手或手动发起安全检查（强制完整 7 项）
4. 「开始实验」→ 命令下发到真实设备
5. 实时接收 DAQ 数据流
6. 自动应用 AI 信号处理（弥散校正等）
7. 跳转到材料分析或生成报告

### 7.5 模式切换的 UX 设计

```
┌──────────────────────────────────────┐
│  虚拟实验室                            │
│                                      │
│  ┌─────────────┐ ┌─────────────┐    │
│  │ 仿真模式 ⚡  │ │ 连接模式 🔌  │    │
│  │ (无需硬件)   │ │ (实时设备)   │    │
│  └─────────────┘ └─────────────┘    │
│                                      │
│  当前选中：[仿真模式]                  │
│  点击切换到：连接模式                  │
└──────────────────────────────────────┘
```

切换时弹窗确认：
- 仿真 → 连接：检查后端连接 → 切换
- 连接 → 仿真：警告"实时数据将断开" → 确认 → 切换

### 7.6 多场耦合的整合

将原 `/multifield` 页面的功能整合到虚拟实验室的**参数面板**中：

```
参数面板：
├── 基础参数
│   ├── 材料
│   ├── 电压
│   ├── 脉宽
│   └── 波形
│
├── ☑ 启用多场耦合
│   ├── 热场
│   │   ├── 温度 (25-1000°C)
│   │   └── 升温速率
│   ├── 力场
│   │   ├── X 轴围压 (0-200 MPa)
│   │   ├── Y 轴围压
│   │   └── Z 轴围压
│   └── 电磁场
│       ├── 强度 (0-120 dB)
│       └── 频率
│
└── ☑ 启用 AI 优化
    └── 优化目标：[最大应力/最大应变率/最大能量/平衡]
```

这样**多场耦合不再是独立页面，而是一个可选的参数维度**，更符合实验逻辑。

### 7.7 AI 优化的整合

将原 `/ai` 页面的 LSTM/WGAN/PPO 三级优化整合为虚拟实验室的**「AI 优化」按钮**：

```
点击「AI 优化」：
    │
    ▼
弹出 Modal 显示三阶段优化进度：
    │
    ├── LSTM 参数扫描 (10 秒)
    ├── WGAN-GP 数据增强 (8 秒)
    └── PPO 精细搜索 (15 秒)
    │
    ▼
显示推荐参数 + 改进百分比
    │
    ▼
[一键应用] / [查看详情]
```

详细的训练动画和神经网络可视化作为「查看详情」的扩展面板，而不是独立页面。

### 7.8 实验工作流（保留并增强）

保留现有 `experimentWorkflow` 状态机，但增强：
- inquiry 阶段支持多场耦合参数询问
- preparation 阶段显示 AI 推荐 + 用户调整
- execution 阶段在连接模式下调用真实设备

### 9.4 五大 Tab 详细设计

#### 9.4.1 Tab 1：原始数据

**功能**：展示来自虚拟实验室或文件导入的原始数据

**元素**：
- 数据来源选择（最近实验 / 历史 / 文件导入）
- 三波波形图（入射/反射/透射）
- 时域信号 + 频域分析（FFT）
- 应力平衡判据可视化（红黄绿）
- 数据元信息（材料、参数、时间）

**操作**：
- 拖入 CSV/JSON/MAT 文件
- 切换不同实验对比
- 导出原始数据

#### 9.4.2 Tab 2：信号处理

**功能**：对原始信号做专业处理

**算法清单**：
- ⭐ **应力波弥散校正**：Pochhammer-Chree 修正
- ⭐ **应变率恒定性补偿**：波形整形分析
- ⭐ **滤波**：低通/带通/小波变换
- ⭐ **基线校正**：去除直流偏移
- ⭐ **同步对齐**：三波时间对齐

**UI**：
- 算法选择 + 参数调节
- 处理前后波形对比
- 处理报告（自动生成）
- 一键应用 / 撤销 / 重置

**实现**：
- 新增 `src/services/signalProcessing/` 模块
- 每个算法独立实现，可链式组合

#### 9.4.3 Tab 3：参数拟合

**功能**：根据处理后的应力-应变曲线拟合本构参数

**支持的本构模型**：
- ⭐ Johnson-Cook
- Cowper-Symonds
- Zerilli-Armstrong
- Power Law
- Bilinear

**UI**：
- 模型选择
- 参数初值（来自材料数据库）
- 拟合算法（最小二乘 / 遗传算法 / 贝叶斯优化）
- 拟合结果 + R²
- 拟合曲线 vs 实验曲线对比
- 残差分析

**实现**：
- 新增 `src/services/constitutiveFitting.ts`
- 集成最小二乘库
- AI 辅助选择最佳模型

#### 9.4.4 Tab 4：对比分析

**功能**：多组实验数据对比

**对比维度**：
- 同一材料不同应变率
- 不同材料同一应变率
- 仿真预测 vs 真实实验
- 不同温度/围压

**UI**：
- 多曲线叠加图
- 6 维性能雷达图
- 关键参数表格
- 趋势图（如 DIF vs 应变率）
- AI 自动总结差异

#### 9.4.5 Tab 5：报告生成

**功能**：一键生成专业实验报告

**报告类型**：

| 类型 | 适用 | 长度 |
|------|------|------|
| 学习总结 | 学生 | 1-2 页 |
| 实验记录 | 日常 | 3-5 页 |
| 研究报告 | 论文准备 | 8-15 页 |
| 完整论文 | 学术发表 | 20+ 页 |

**报告内容**：
- 实验目的、装置、原理
- 参数设置
- 数据采集与处理
- 拟合结果
- 结论与建议
- 数据附录
- 可复现 Python 代码

**导出格式**：
- PDF（含图表）
- Word（可继续编辑）
- LaTeX（论文模板）
- Markdown（轻量）
- JSON（数据备份）

### 9.5 与 AI 助手的深度集成

材料分析页面是 AI 助手的**重点驻留区域**：

| 场景 | AI 行为 |
|------|--------|
| 用户加载数据 | AI 主动总结："这是 5A06 在 1500/s 下的数据，应力平衡良好" |
| 用户处理信号 | AI 推荐："建议先做弥散校正，原始波形高频段有失真" |
| 用户拟合参数 | AI 评估："拟合 R² = 0.94，建议尝试 Cowper-Symonds 模型" |
| 用户对比数据 | AI 解读："这两组数据应变率差 5 倍，DIF 增长符合预期" |
| 用户生成报告 | AI 引导："你想要哪种类型的报告？" |

### 10.5 安全与可靠性

#### 10.5.1 通信安全

- WebSocket 使用 WSS（TLS 加密）
- API 接口使用 JWT 认证
- 关键操作需要二次确认 token
- 日志审计所有命令下发

#### 10.5.2 故障恢复

- WebSocket 断线自动重连（指数退避）
- 命令幂等性设计
- 心跳检测设备状态（5 秒一次）
- 紧急停止有独立物理通道

#### 10.5.3 数据一致性

- 所有实验数据先写入数据库再返回
- 使用事务保证原子性
- 定期备份（每日自动）
