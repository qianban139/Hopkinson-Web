# 围压 PID 闭环伺服

> 实现位置：`src/services/pidController.ts` + `src/services/confiningPressureSimulator.ts`
> UI 入口：MultiField 页面 → "PID 闭环伺服调控 (围压)" 卡片

## 被控对象模型

将"伺服阀 + 液压缸 + 试样腔"建模为带时滞的二阶系统：

```
τ² p̈ + 2ζτ ṗ + p = K · u(t - Td) + d(t)
```

| 符号 | 含义 | 默认值 |
|---|---|---|
| `p` | 实际围压 (MPa) | — |
| `u` | 控制器输出（归一化 0–1，伺服阀开度） | — |
| `K` | 静态增益（伺服阀全开稳态压力） | 200 MPa |
| `τ` | 时间常数 | 0.35 s |
| `ζ` | 阻尼比（< 1 欠阻尼，便于演示超调） | 0.45 |
| `Td` | 纯时滞（伺服阀响应延迟） | 0.06 s |
| `d` | 扰动（试样破碎瞬间泄压等） | t=3.5s 注入 −10 MPa / 0.4s |

数值积分：显式二阶欧拉，dt = 0.01 s（100 Hz 采样）。
时滞用环形缓冲实现：`uBuffer.push(u); uDelayed = uBuffer.shift()`。

## PID 控制律

标准位置式（**audit 修复后**：PID 直接输出归一化伺服开度 `u ∈ [0, 1]`）：

```
u(t) = Kp·e(t) + Ki·∫e(τ)dτ + Kd·de(t)/dt
e(t) = setpoint(t) - measured(t)   单位 MPa
u(t) ∈ [0, 1]                       伺服阀开度
```

输出夹紧到 `[outMin=0, outMax=1]` + **方向感知 anti-windup**（仅当本步积分增量与饱和方向同向时回退，允许积分回退以便饱和退出）。

D 项基于测量值（derivative on measurement，避免设定值阶跃的微分冲击）+ α=0.3 低通滤波。

## 三档预设

> 增益单位：`MPa → 开度` 标度（误差 50 MPa 时 P 项贡献量见下表）

| 预设 | Kp | Ki | Kd | P 贡献 (e=50) | 特征 |
|---|---|---|---|---|---|
| 保守 | 0.005 | 0.008 | 0.0003 | 0.25 (25%) | 无超调，响应慢（~2 s 才到目标） |
| 经典 ZN | 0.015 | 0.025 | 0.0008 | 0.75 (75%) | 响应快（~0.7 s），轻微超调 |
| 激进 | 0.030 | 0.050 | 0.0015 | 1.50 (>1, 饱和) | 极快，明显超调与震荡，anti-windup 介入 |

旧版（v1.1 audit 修复前）增益分别是上述值的 ~200 倍，因 `outMax=K` 导致与归一化开度耦合，已修复。

## 性能指标

| 指标 | 定义 |
|---|---|
| 超调 % | `max(measured - target) / target × 100` |
| 稳态误差 | 最后 15% 时间内 `measured - target` 平均 |
| 2% 调节时间 | 最后一次越出 `target ± 2%` 带的时刻 |
| 跟踪 RMSE | `sqrt(mean((measured - setpoint)²))` |

## 局限（必读）

1. **纯前端仿真，无真实硬件接入**：用于教学演示，**不代表**实际液压伺服阀的响应。
2. **位置式 PID + anti-windup**：audit PID-1 修复后已实现方向感知 anti-windup（饱和方向与本步积分增量同向时冻结），但仍是位置式而非增量式 — 大动态范围切换时仍可能瞬时偏差。
3. **K = 200 MPa 偏高**：实际典型 SHPB 围压系统 K 在 100–150 MPa，此处选 200 是为演示需要让 50 MPa 目标处于 25% 工作点。
4. **settlingTime 边界**：若 measured 始终在 ±2% 带内，evalResponse 返回 0（语义为"立刻稳态"，audit PID-5 已加注释说明）。
